## Key Decisions Made

1. **Email Provider:** Resend - Modern API, good deliverability, React Email support
2. **SMS Provider:** Twilio - Industry standard, reliable, good documentation
3. **Push:** Web Push API with VAPID - Native browser support, no third-party service needed
4. **Timezone:** date-fns-tz - Lightweight, works well with existing date handling

---

## Architecture Notes

- Notification service will be centralized in `src/lib/notifications/`
- Each channel (email, push, SMS) has its own module in `channels/`
- Triggers are event-based, invoked by cron jobs or system events
- User preferences stored in new `notification_preferences` table
- Queue system for retry logic and scheduling
