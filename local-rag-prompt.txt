قم ببناء تطبيق Flask يعمل كـ RAG محلي بالكامل — لا يتم إرسال أي بيانات خارج الجهاز نهائياً.

المتطلبات:

1. المزامنة:
   - استخدم watchdog لمراقبة مجلد company_docs
   - عند إضافة أو تعديل أي ملف (PDF, DOCX, TXT)، قم بـ:
     أ. استخراج النص من الملف (PyPDF2 / python-docx / plain text)
     ب. تقسيم النص إلى chunks (500 كلمة مع overlap 50 كلمة)
     ج. توليد embeddings محلياً عبر Ollama باستخدام نموذج nomic-embed-text
     د. تخزين الـ embeddings في قاعدة ChromaDB المحلية
   - عند حذف ملف، قم بإزالة الـ vectors المرتبطة به من ChromaDB

2. المحادثة:
   - واجهة ويب بسيطة (HTML + CSS) ترسل أسئلة العميل إلى Flask backend
   - الـ backend يقوم بـ:
     أ. توليد embedding للسؤال عبر Ollama (nomic-embed-text)
     ب. البحث عن أقرب chunks في ChromaDB (similarity search, top 5)
     ج. إرسال السؤال + السياق المسترجع إلى نموذج qwen3:1.7b عبر Ollama API
        (http://localhost:11434/api/chat)
     د. إرجاع الإجابة للعميل — يجيب من الوثائق فقط

3. القيود:
   - يُمنع تماماً استخدام LangChain أو LlamaIndex أو Pinecone أو أي مكتبات RAG خارجية
   - يُمنع استخدام أي خدمات سحابية (OpenAI, Azure, Google, AWS)
   - الاعتماد 100% على: Python + Flask + Ollama + ChromaDB
   - جميع العمليات تتم محلياً على الجهاز فقط

4. المكتبات المسموحة:
   - flask (واجهة الويب)
   - chromadb (قاعدة vectors محلية)
   - watchdog (مراقبة الملفات)
   - requests (للاتصال بـ Ollama على localhost فقط)
   - PyPDF2 (استخراج نص PDF)
   - python-docx (استخراج نص DOCX)

5. نماذج Ollama المطلوبة:
   - qwen3:1.7b → للمحادثة وتوليد الإجابات
   - nomic-embed-text → لتوليد الـ embeddings

6. التشغيل:
   - تأكد أن Ollama يعمل: ollama serve
   - تأكد من تحميل النماذج: ollama pull qwen3:1.7b && ollama pull nomic-embed-text
   - شغّل التطبيق: python app.py
   - افتح المتصفح: http://localhost:5000
